## II. Fundamentación y Justificación

Esta asignatura aporta al perfil del Ingeniero Informático la formación de personal con un dominio sobre el proceso de verificación y validación con un enfoque hacia la calidad del software.

Los aspectos principales para considerar son las revisiones y las pruebas como parte del ciclo de vida del software que se utilizan para detectar fallos en el sistema.

La verificación y validación del software puede llegar a suponer el 40% del tiempo dedicado al desarrollo de un producto software. Por tanto, resulta fundamental que los ingenieros de software conozcan las técnicas para la verificación y validación de software, así como el manejo de herramientas para el Testing.

## IV. Competencias de la Asignatura

### 4.1. Competencia Global

Maneja y aplica técnicas para verificaciones y validaciones al software, considerando los aspectos de revisión y pruebas como parte del ciclo de vida del software.

### 4.2. Competencias Específicas

Al finalizar la asignatura, el estudiante será capaz de:

1. Mantener y evaluar servicios y sistemas software que satisfagan todos los requisitos del usuario y se comporten de forma fiable y eficiente, sean asequibles y seguros.
2. Conocer la importancia del testing del software y el aseguramiento de calidad.
3. Desarrollar dentro del proceso de ingeniería del software cuándo y cómo se deben realizar las tareas de pruebas y aseguramiento de calidad.
4. Analizar las estrategias, métodos y técnicas de diseño de testing para realizar pruebas del software.
5. Diseñar elementos para poder organizar, planificar, estimar y controlar las pruebas.
6. Manejar herramientas de soporte a las pruebas, analizar diferentes alternativas del mercado y su orientación.
